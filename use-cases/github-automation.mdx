## Use Case 1: Automated PR Creation & Review

### Scenario

Your code analysis agent detects a bug in production. It needs to:

1. Create a hotfix PR with the solution
2. Assign the PR to the on-call engineer
3. Post a review comment explaining the fix
4. Notify the team on Slack

### Implementation

```javascript
const agent = new AgentCorex({
  apiKey: process.env.AGENT_COREX_API_KEY
});

async function handleProductionBug(bugReport) {
  // Step 1: Find the right tools
  const tools = await agent.retrieveTools({
    query: "Create GitHub PR, assign to engineer, add review comment, notify team",
    topK: 5
  });

  console.log('Available tools:', tools);
  // [
  //   {name: "create-pull-request", score: 0.99},
  //   {name: "request-pull-request-review", score: 0.96},
  //   {name: "post-comment", score: 0.94},
  //   {name: "send-slack-message", score: 0.93}
  // ]

  // Step 2: Create PR
  const prResult = await agent.executeTool({
    toolName: 'create-pull-request',
    params: {
      repository: 'company/main-app',
      title: `HOTFIX: ${bugReport.title}`,
      body: `**Issue:** ${bugReport.description}\n\n**Root Cause:** ${bugReport.rootCause}\n\n**Solution:** ${bugReport.fix}`,
      fromBranch: 'hotfix/bug-123',
      toBranch: 'main'
    }
  });

  // Step 3: Request review
  await agent.executeTool({
    toolName: 'request-pull-request-review',
    params: {
      repository: 'company/main-app',
      pullNumber: prResult.pullNumber,
      reviewers: ['oncall-engineer']
    }
  });

  // Step 4: Post comment
  await agent.executeTool({
    toolName: 'post-comment',
    params: {
      repository: 'company/main-app',
      pullNumber: prResult.pullNumber,
      body: `🤖 This PR was auto-generated by AI Bug Detector.\n\n**Confidence:** 98%\n**Fix Type:** Hotfix\n**Risk Level:** Low`
    }
  });

  // Step 5: Notify team
  await agent.executeTool({
    toolName: 'send-slack-message',
    params: {
      channel: '#incidents',
      message: `🚨 Production bug fixed!\nPR: ${prResult.pullUrl}\nReady for urgent review.`
    }
  });

  return prResult;
}
```

### Result

**Before:**
- 10 minutes for engineer to manually create PR, review, comment
- Risk of delays during incidents

**After:**
- 2 seconds for agent to do everything
- Engineer notified immediately
- Faster incident response

## Use Case 2: Intelligent Issue Triage

### Scenario

Your support agent receives a new GitHub issue. It needs to:

1. Analyze the issue content
2. Assign the right label (bug/feature/question)
3. Assign to the right team
4. Post a response template
5. Track in Jira if needed

### Query for Tool Discovery

```javascript
const tools = await agent.retrieveTools({
  query: "Analyze GitHub issue, add labels, assign to team, post response comment",
  topK: 5
});
```

### Tool Execution

```javascript
async function triageIssue(issue) {
  // Analyze issue
  const analysis = analyzeIssue(issue.body);

  // Add label
  await agent.executeTool({
    toolName: 'add-issue-label',
    params: {
      repository: 'company/main-app',
      issueNumber: issue.number,
      labels: [analysis.type, analysis.severity]
      // e.g., ['bug', 'critical']
    }
  });

  // Assign to team
  await agent.executeTool({
    toolName: 'assign-issue',
    params: {
      repository: 'company/main-app',
      issueNumber: issue.number,
      assignees: [analysis.assignee]
      // e.g., ['backend-team']
    }
  });

  // Post response
  const responseTemplate = TEMPLATES[analysis.type];
  await agent.executeTool({
    toolName: 'post-comment',
    params: {
      repository: 'company/main-app',
      issueNumber: issue.number,
      body: responseTemplate
    }
  });

  // Create Jira ticket if critical bug
  if (analysis.severity === 'critical' && analysis.type === 'bug') {
    await agent.executeTool({
      toolName: 'create-jira-issue',
      params: {
        project: 'PLATFORM',
        title: issue.title,
        description: issue.body,
        priority: 'Critical'
      }
    });
  }
}
```

| **Consistency** | Variable | 100% consistent |
| **Mislabeling** | 20% | <1% |
| **Response Time** | 1-2 hours | Immediate |

## Use Case 3: Automated Release Management

### Scenario

Your deployment agent needs to:

1. Find all merged PRs since last release
2. Generate release notes from PR descriptions
3. Create GitHub release
4. Tag the commit
5. Trigger CI/CD pipeline

### Implementation

```javascript
async function createRelease(version) {
  const tools = await agent.retrieveTools({
    query: "Get merged PRs, create GitHub release, tag commits, trigger deployment",
    topK: 5
  });

  // Get merged PRs
  const prs = await agent.executeTool({
    toolName: 'list-pull-requests',
    params: {
      repository: 'company/main-app',
      state: 'merged',
      since: lastReleaseDate,
      limit: 100
    }
  });

  // Generate release notes
  const releaseNotes = generateReleaseNotes(prs);

  // Create GitHub release
  const release = await agent.executeTool({
    toolName: 'create-release',
    params: {
      repository: 'company/main-app',
      tagName: `v${version}`,
      targetBranch: 'main',
      name: `Release ${version}`,
      body: releaseNotes,
      draft: false,
      prerelease: false
    }
  });

  // Trigger deployment
  await agent.executeTool({
    toolName: 'trigger-github-action',
    params: {
      repository: 'company/main-app',
      workflow: 'deploy.yml',
      ref: `v${version}`
    }
  });

  // Notify
  await agent.executeTool({
    toolName: 'send-slack-message',
    params: {
      channel: '#releases',
      message: `🎉 Released v${version}\n${release.url}`
    }
  });
}
```

## Use Case 4: Code Review Automation

### Scenario

Your code review agent analyzes every new PR and:

1. Checks for common issues (linting, security)
2. Posts automated review comments
3. Requests human review if needed
4. Updates PR status check

### Implementation

```javascript
async function reviewPullRequest(pr) {
  const tools = await agent.retrieveTools({
    query: "Analyze code, post review comments, check PR status"
  });

  // Get changed files
  const files = await agent.executeTool({
    toolName: 'get-pull-request-files',
    params: {
      repository: pr.repository,
      pullNumber: pr.number
    }
  });

  // Analyze each file
  const issues = [];
  for (const file of files) {
    const analysis = analyzeCode(file);
    issues.push(...analysis);
  }

  // Post review comments
  for (const issue of issues) {
    await agent.executeTool({
      toolName: 'post-review-comment',
      params: {
        repository: pr.repository,
        pullNumber: pr.number,
        commitId: issue.commitId,
        path: issue.filePath,
        line: issue.lineNumber,
        body: `⚠️ ${issue.message}\n\n**Suggestion:** ${issue.suggestion}`
      }
    });
  }

  // Submit review
  const review = issues.length > 0 ? 'REQUEST_CHANGES' : 'APPROVE';
  await agent.executeTool({
    toolName: 'submit-review',
    params: {
      repository: pr.repository,
      pullNumber: pr.number,
      event: review,
      body: `Automated code review complete. Found ${issues.length} issue(s).`
    }
  });
}
```

## Best Practices

### 1. Meaningful PR Titles & Descriptions

```javascript
// ❌ Bad
title: "Update"
body: "Fixed stuff"

// ✅ Good
title: "HOTFIX: Resolve payment processing race condition"
body: `**Issue:** #1234\n\n**Root Cause:** Race condition in concurrent payment processing\n\n**Solution:** Added mutex lock around transaction processing\n\n**Testing:** Added regression test\n\n**Risk:** Low - only affects concurrent payments`
```

### 2. Use Labels Consistently

```javascript
// Standard labels for sorting
['type:bug', 'type:feature', 'type:refactor']
['severity:critical', 'severity:high', 'severity:low']
['status:blocked', 'status:in-progress', 'status:ready-review']
```

### 3. Monitoring & Alerts

```javascript
// Alert if PR hasn't been reviewed in 2 hours
if (pr.createdAt < now - 2h && pr.reviewers.length === 0) {
  await agent.executeTool({
    toolName: 'send-slack-message',
    params: {
      channel: '#engineering',
      message: `⏱️ PR waiting for review: ${pr.url}`
    }
  });
}
```
