# Tool Discovery & Retrieval API

Powerful semantic search to find the right tools for any task.

---

## Overview

The Tool Discovery API provides three progressive levels of tool searching:

1. **Basic Retrieval** (`/retrieve_tools`) - Simple semantic search
2. **Advanced V2/V3** (`/v2/retrieve_tools`) - Hybrid with capability filtering
3. **Registry Lookup** (`/tools/registry`) - Browse all available tools

---

## GET /retrieve_tools

**Semantic tool search with optional filtering**

### Authentication
Optional (returns more detailed results with API key)

### Query Parameters

| Parameter | Type | Required | Default | Max |
|-----------|------|----------|---------|-----|
| `query` | string | Yes | - | 500 chars |
| `top_k` | integer | No | 5 | 50 |

### Request Examples

**Basic Search:**
```bash
curl "https://api.agent-corex.com/retrieve_tools?query=create+github+pull+request" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**With Result Limit:**
```bash
curl "https://api.agent-corex.com/retrieve_tools?query=deploy&top_k=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Response Format

```json
{
  "tools": [
    {
      "name": "create_pull_request",
      "server": "github-mcp",
      "score": 0.98,
      "description": "Create a new pull request on GitHub",
      "category": "version-control",
      "tags": ["github", "git", "pr"],
      "inputSchema": {
        "type": "object",
        "properties": {
          "repository": { "type": "string", "description": "Repo in owner/repo format" },
          "title": { "type": "string" },
          "from_branch": { "type": "string" },
          "to_branch": { "type": "string" }
        },
        "required": ["repository", "title", "from_branch", "to_branch"]
      }
    }
  ],
  "query_time_ms": 45,
  "total_tools_searched": 156
}
```

### Error Responses

**Invalid Query (400):**
```json
{
  "error": "Query parameter required and must be between 1-500 characters"
}
```

**Rate Limited (429):**
```json
{
  "error": "Rate limit exceeded. Reset in 3600 seconds."
}
```

## GET /v2/retrieve_tools

**Advanced hybrid retrieval with capability classification**

### Authentication
Optional

### Query Parameters

| Parameter | Type | Required | Default | Notes |
|-----------|------|----------|---------|-------|
| `query` | string | Yes | - | User query |
| `top_k` | integer | No | 5 | Range: 1-20 |
| `use_hybrid_v3` | boolean | No | true | Use V3 algorithm |

### Key Features

- **Capability Filtering** - Classifies query intent automatically
- **Query Rewriting** - Shows normalized query for transparency
- **MCP Recommendations** - Suggests which servers to install
- **Confidence Scores** - Shows how confident the classification is

### Request Examples

**Standard Advanced Search:**
```bash
curl "https://api.agent-corex.com/v2/retrieve_tools?query=deploy+to+AWS&top_k=5&use_hybrid_v3=true" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Explicit Server Focus:**
```bash
curl "https://api.agent-corex.com/v2/retrieve_tools?query=manage+kubernetes+clusters&top_k=10"
```

### Response Format

```json
{
  "tools": [
    {
      "name": "deploy_to_aws",
      "server": "aws-mcp",
      "score": 0.96,
      "description": "Deploy application to AWS",
      "inputSchema": { /* ... */ }
    }
  ],
  "selected_capability": "deployment",
  "capability_confidence": 0.95,
  "query_rewritten": "Deploy application to AWS cloud platform",
  "recommended_mcps": [
    {
      "name": "aws-mcp",
      "reason": "Primary deployment tools"
    },
    {
      "name": "terraform-mcp",
      "reason": "Infrastructure as code management"
    }
  ],
  "query_id": "q_abc123def456"
}
```

### Capability Classifications

The API automatically classifies queries into:

- **deployment** - Cloud/environment deployment
- **infrastructure** - Infrastructure management
- **monitoring** - Observability/logging
- **version-control** - Git/GitHub operations
- **ci-cd** - Pipeline/automation
- **databases** - Database operations
- **communication** - Slack/email/messaging
- **development** - Code/IDE tools
- **devops** - Operations tooling
- **other** - Uncategorized

## Search Tips

### Writing Effective Queries

**❌ Bad Queries:**
- "stuff"
- "help"
- "do things"

**✅ Good Queries:**
- "Create a GitHub pull request"
- "Deploy to AWS and notify Slack"
- "Monitor logs and create alert"

### Query Optimization

1. **Be Specific** - Include tool names or services
2. **Use Action Verbs** - "create", "deploy", "manage", "monitor"
3. **Specify Services** - "GitHub", "AWS", "Slack"
4. **Include Context** - "production", "staging", "all regions"

### Multi-Step Workflows

For complex workflows, chain multiple queries:

```bash
# Step 1: Find deployment tools
curl "https://api.agent-corex.com/retrieve_tools?query=deploy+to+AWS"

# Step 2: Find notification tools
curl "https://api.agent-corex.com/retrieve_tools?query=send+Slack+notification"

# Step 3: Execute both in sequence
curl -X POST "https://api.agent-corex.com/execute_tool" \
  -d '{"tool": "deploy_to_aws", "arguments": {...}}'
curl -X POST "https://api.agent-corex.com/execute_tool" \
  -d '{"tool": "send_slack_message", "arguments": {...}}'
```

## Response Scoring

Tools are ranked by relevance score (0.0 to 1.0):

| Score | Meaning |
|-------|---------|
| 0.90+ | Highly relevant - exact match |
| 0.75-0.89 | Very relevant - good match |
| 0.60-0.74 | Relevant - related match |
| <0.60 | Marginal - use with caution |

## Caching Strategy

Tool lists don't change frequently. Consider caching:

```javascript
// Cache tools for 1 hour
const CACHE_TTL = 3600000; // 1 hour

async function getToolsWithCache(query) {
  const cacheKey = `tools_${query}`;
  const cached = localStorage.getItem(cacheKey);
  
  if (cached && Date.now() - JSON.parse(cached).timestamp < CACHE_TTL) {
    return JSON.parse(cached).data;
  }
  
  const tools = await fetch(`/retrieve_tools?query=${query}`);
  localStorage.setItem(cacheKey, JSON.stringify({
    data: tools,
    timestamp: Date.now()
  }));
  
  return tools;
}
```

## Error Handling

```javascript
async function safeRetrieveTools(query) {
  try {
    const response = await fetch(`/retrieve_tools?query=${query}`, {
      headers: { 'Authorization': `Bearer ${apiKey}` }
    });
    
    if (!response.ok) {
      if (response.status === 429) {
        throw new Error('Rate limited - retry in 60 seconds');
      } else if (response.status === 400) {
        throw new Error('Invalid query');
      }
    }
    
    return await response.json();
  } catch (error) {
    console.error('Tool retrieval failed:', error);
    return { tools: [] };
  }
}
```
